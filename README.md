Репозиторий содержит необходимый набор файлов для выполнения практической работы по логическому синтезу и топологическому проектированию блока цифровой микросхемы в используя open-source САПР в среде OpenLane/LibreLane.

В данной работе мы будем не просто запускать автоматизированный маршрут с различными параметрами, а более детально посмотрим на основные этапы маршрута, их входные и выходные данные, соответствующие команды САПР. Это позволит получить реальное представление о работе тополога систем на кристале.

Описание заданий доступно по [ссылке](https://vlsi-be-labs.readthedocs.io/ru/latest/index.html). Описание не включает в себя теоретический материал на данный момент не включает теоретический материал.

Цель работы – приобретение базовых навыков логического синтеза и топологического проектирования блоков цифровых схем с помощью САПР.

Для достижения цели работы предлагается пройти указанные этапы маршрута для простейшей схемы, представленной на языке описания аппаратуры с использованием реальной открытой библиотеки. 

## Используемое ПО

Для выполнения работы необходимы следующие САПР:

- [Yosys](https://github.com/YosysHQ/yosys) - для логического синтеза;
- [OpenRoad](https://github.com/The-OpenROAD-Project) - для разработки топологии;
- [Magic](http://opencircuitdesign.com/magic/) - для генерации финального GDS;
- [KLayout](https://klayout.de/) - для просмотра GDS и LEF файлов.

Ссылки ведут на публичные репозитории или сайты САПР, где приведены инструкции для установки, документация, исходный код. Следует отметить, что данные САПР поддерживаются только дистрибутивами Linux, либо macOS. Для работы с Windows вам потребуется предварительная установка виртуальной машины Linux. Непосредственная установка каждого САПР рекомендуется в случае, если вы планируете углубленное изучение темы, выходящее за рамки данной работы. Мы же будем использовать более простой способ, а именно работу внутри пакета [OpenLane](https://github.com/The-OpenROAD-Project/OpenLane) (на данный момент разработка OpenLane приостановлена, проектом приемником стал [LibreLane](https://github.com/librelane/librelane) - рекомендуется к использованию). Основная задача OpenLane/LibreLane - полностью автоматизированный синтез GDS из RTL. В результате многие процессы, такие как разработка floorplan остаются скрытыми.
